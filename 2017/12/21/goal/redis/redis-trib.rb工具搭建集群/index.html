redis-trib安装

redis-trib.rb是采用ruby实现的redis集群管理工具。内部通过cluster相关命令帮我们简化集群创建，检查，槽迁移和均衡等常见运维操作。使用之前需要安装ruby依赖环境，redis作者用这个工具有很多我们值得借鉴学习的方法和习惯(像集群水平扩容/缩容 集群创建等可以避免知识误区)。从以下下部分开始使用讲解:

组件依赖

快速搭建集群

准备节点

准备至少3个redis节点,启动节点:

redis集群创建

创建master集群: redis-trib create ${host}:${port} …

创建master-slave集群： redis-trib.rb create –replicas 1 ${host}:${port} …

集群完整性检查： redis-trib.rb check 127.0.0.1:6380

集群完整性指所有的槽都分配到存活的主节点上，当16384个槽中有任意一个没有分配给节点表示集群不完整。

集群状态 ：redis-trib.rb info 127.0.0.1:6380

新增节点

启动一个新redis节点6383，

添加新节点:redis-trib.rb add-node ${newHost:newPort} ${existHost:existPort}

槽自动迁移

槽迁移步骤:

  • 1):对目标节点发送cluster setslot {slot} importing {sourceNodeId}命令，让目标节点准备导入槽的数据。
  • 2:对源节点发送clusgter setslot {slot} migrating {targetNodeId}命令，让源节点准备迁出槽的数据。
  • 3:源节点循环执行cluster getkeysinslot {slot} {count}命令，每次获取count个属于槽的健。
  • 4:在源节点上执行migrate {targetIp} {targetPort} “” 0 {timeout} keys {keys…} 命令把获取的键通过管道(pipeline)机制批量迁移到目标节点,批量迁移版本的migrate命令在Redis3.0.6版本提供，之前的migrate命令只能单个键迁移。批量键迁移对于大量key的场景将极大降低节点之间网络IO次数。
  • 5:重复执行步骤3和步骤4直到槽下所有的键数据迁移到目标节点。
  • 6:向集群内所有主节点发送cluster setslot {slot} node {targetNodeId}命令，通知槽分配给目标节点。为了保证槽节点映射变更及时传播，需要遍历发送给所有主节点更新被迁移的槽指向新节点。

redis-trib自动迁移计划:

槽迁移 : redis-trib.rb reshard ${host:port}

rebalance节点槽分配

redis-trib.rb rebalance 127.0.0.1:6383
