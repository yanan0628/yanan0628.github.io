问题分析

集群节点状态变化

集群migrate迁移过程

步骤:

  • 1.设置目标节点为导入状态importing
  • 2.设置源节点为迁移状态migrating
  • 3.遍历源节点${slot}的键，migrate 单键迁移操作，只至当前slot迁移完毕(为什么不批量迁移?redis在migrate批量键迁移存在bug)
  • 4.对每一个master节点更新 目标节点绑定当前slot信息(问题在于获取集群master节点不准确，参考下节)
    通过日志和节点集群状态观察，之前集群master节点状态一致，新加master节点10.10.79.150:6407 key全部迁移成功，只是未更新集群信息。

重新获取集群所有master节点

  • 重新初始化jediscluster, hosts节点为10个，但是重新从JedisClusterInfoCache获取nodes节点个数为8,问题出在这里： jedisCluster.getConnectionHandler().getNodes().values()

  • 查看jedisClusterInfoCache源码，setupNodeIfNotExist(targetNode)：变更 nodes(Map)、slots(Map)关系

如果新加入节点且未被分配slots,新节点节点不会更新到集群 node cache，所以这里水平扩容需要注意新节点不会set到nodes的问题，最终会导致节点bind slot失败，需要手动设置。
